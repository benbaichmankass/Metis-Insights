### M19 — AI Model Strategy (new model types) (phased proposal)

> **Status:** 📋 PROPOSED 2026-07-01 (research/design session) — **PROPOSE-ONLY**. No
> `src/`, `config/`, `ml/`, or live-path file changed by the proposal. Full design
> (reframe → current-state inventory → the capability ladder → compute-tier
> economics → the "reads-everything" encoder → trade-offs → gates → phase
> checklist): [`docs/research/ai-model-strategy-roadmap-2026-07-01.md`](docs/research/ai-model-strategy-roadmap-2026-07-01.md).
> Successor to the largely-complete M14 optimization program. Every model type
> graduates observe-only through `candidate → shadow → advisory`; nothing
> influences a live order until a backtest A/B passes and the operator approves.

**The reframe:** every model we run today is a *narrow tabular classifier over
hand-engineered features* (46 manifests: LightGBM regime/vol + trade-outcome +
setup-quality heads, one HMM, baselines) — they differ in target/symbol, not in
*kind*. M19 opens new model **types**. And the binding constraint is **labels
(~350 real trades), not compute** (the free 1-OCPU trainer clears the daily
cycle in <1h) → **self-supervised representation learning (label-free) is the
flagship**, and deep models enter as *encoders feeding the existing LightGBM
heads*, never end-to-end replacements.

**Compute tiers:** Tier 0 = **$0** free CPU (LightGBM/HMM/GMM + **frozen**
pretrained-TSFM CPU embeddings) — start now; Tier 1 = **~$10/mo** gated spot-GPU
bursts (deep sequence models + in-house self-supervised encoder v0; train on
GPU, **export inference to CPU/ONNX**) — spec'd now, spending gated on an operator
go-signal; Tier 2 = larger long-term spend (multi-task foundation encoder + RL).
