### Sizing logic

All three concrete strategy builders resolve `qty` the same way:

| Strategy | Sizing code | File : line |
|---|---|---|
| `killzone` | `float(settings.get("MAX_QTY", ...) or 1)` | `pipeline.py:96` |
| `vwap` | `float(settings.get("MAX_QTY", ...) or 1)` | `pipeline.py:123` |
| `breakout_confirmation` | ATR path computed but **overwritten** by `qty = fallback_qty` unconditionally | `pipeline.py:190‚Äì194` |

**Risk sizing gap:** `breakout_model_signal_builder` (pipeline.py:185‚Äì207) calculates `atr` from `model_signal` and intends to size by ATR, but lines 193‚Äì194 always assign `qty = fallback_qty` regardless of `atr > 0`. The ATR branch is dead code.

### Risk guards

Three hard risk guards live in `safe_place_order()` (orders.py:77‚Äì109). All three raise `ValueError`, which aborts the order unconditionally:

| Guard | Env var(s) checked | orders.py lines |
|---|---|---|
| Position notional | `MAX_POSITION_USD`, order price | 78‚Äì87 |
| Daily loss | `MAX_DAILY_LOSS_USD`, `CURRENT_DAILY_LOSS_USD` | 89‚Äì98 |
| Open positions | `MAX_OPEN_POSITIONS`, `CURRENT_OPEN_POSITIONS` | 100‚Äì109 |

**Gap:** `CURRENT_DAILY_LOSS_USD` and `CURRENT_OPEN_POSITIONS` are read from `settings` (not computed from the exchange or DB). If the caller does not inject current values, the guards silently pass. There is no code in `run_pipeline()` (pipeline.py:309‚Äì394) that populates these counters before calling `safe_place_order`.

Two additional safety gates operate as soft blocks: `DRY_RUN` (orders.py:121, defaults to `"true"`) and `ALLOW_LIVE_TRADING` (orders.py:128, defaults to `"false"`). Both default safe.

Kill-switch: `HALT_FLAG_PATH = "/tmp/trader_halt.flag"` (pipeline.py:7). Checked at pipeline.py:354 before `safe_place_order` is reached.

### Strategy multiplexing

Controlled entirely by the `STRATEGY` env var read at `run_pipeline()` entry (pipeline.py:318). Only **one** strategy builder executes per tick:

```python
# pipeline.py:268
STRATEGIES = ["breakout_confirmation", "vwap"]   # multiplexed priority order

# pipeline.py:318‚Äì329  (run_pipeline dispatch)
if   strategy_name == "multiplexed": builder = multiplexed_signal_builder
elif strategy_name == "vwap":        builder = vwap_signal_builder
elif strategy_name == "breakout":    builder = breakout_model_signal_builder
else:                                builder = killzone_signal_builder  # default
```

`multiplexed_signal_builder` (pipeline.py:277‚Äì306) iterates `STRATEGIES` in order; first `side ‚àà {buy, sell}` with `qty > 0` wins. Strategies that raise are logged and skipped (pipeline.py:294‚Äì296). If all return flat, returns `side=none` (pipeline.py:305‚Äì306).

**Gap:** `multiplexed` is not the default ‚Äî `STRATEGY` defaults to `"killzone"` (pipeline.py:318), which is not in `_STRATEGY_BUILDERS` as "killzone" but is the `else` branch. `killzone` is also not included in `STRATEGIES` (pipeline.py:268), so it is never tried in multiplexed mode.
