# MB-20260701-001 — gate 4: vol-gate operating-threshold MONEY A/B (2026-07-17)

## Verdict

**HONEST NEGATIVE for the threshold change — keep the live 0.005 vol-gate.** The
denser-label 0.004 head does NOT beat the shipped 0.005 head on the walk-forward
money A/B: it is **worse on aggregate net PnL (+$425 vs +$502, −$77)** and has
**equal-or-worse maxDD in every one of the four folds**. Gate 2 established that
0.004 is a materially better *classifier* (f1_volatile 0.341 vs 0.249, macro_f1
0.629 vs 0.598) — gate 4 shows that lift **does not convert to a money edge** as the
vol-gate's routing verdict. This resolves MB-20260701-001: **the vol-gate threshold
stays 0.005; the flip is declined on the money evidence.**

## Why this is the decisive gate

MB-20260701-001 asked whether the live BTC-15m vol-gate (which reads the advisory
head's `P(volatile)` and is data-starved at the 0.005 label's 4.6% volatile base
rate) would route better at a denser ~0.004 label. Gate 1 reconciled the
threshold→base-rate mapping; gate 2 pinned the operating curve (0.004 is a better
classifier). But **the gate is a live money-routing switch, not a classifier** — a
denser label changes *when* the OFF-cells gate, which has a direct P&L consequence
only a backtest measures (gate 2 § "honest bounds" spelled this out). Gate 4 is that
backtest, on the same bar the 2026-06-27 go-live A/B cleared.

## Method (Tier-1 research; the live 0.005 threshold was never touched)

**Matched-sibling design** — the two arms are `btc-regime-15m-lgbm-vt005-pin-v1`
(0.005, dataset v515) and `btc-regime-15m-lgbm-vt004-pin-v1` (0.004, dataset v514),
built **identically** (same 7 features, 60d recency half-life, purged 5-fold
walk-forward CV, inverse-base-rate class weight) and differing **only** in the
`vol_threshold` label. So the per-fold money delta isolates the threshold change,
free of the data-window/build confounds a production-v2 baseline would carry.

Both heads register-trained on the trainer (reproducing the gate-2 curve to the
digit: 0.005 f1_volatile 0.249 / macro_f1 0.598; 0.004 f1_volatile 0.341 / macro_f1
0.629), then promoted candidate→shadow (autonomous, below the operator-gated
shadow→advisory line; shadow never influences a live order) so
`backtest_system.py --ml-stage shadow` can load them. Walk-forward:
`scripts/ml/walkforward_vol_threshold.sh` → `scripts/ml/gate4_vol_threshold_run.sh`,
`--regime-router on --vol-verdict ml` over the candidate `trend_vol` policy
(`regime_policy_trend_vol_candidate-2026-06-27.yaml`), roster
`trend_donchian,squeeze_breakout_4h,htf_pullback_trend_2h`, `data/backtest_BTCUSDT_5m.csv`,
four consecutive non-overlapping yearly folds (implicit purge between folds).

## Result — per fold (net PnL, maxDD, return/DD)

| Fold (start→end) | 0.005 net / maxDD / ret-DD | 0.004 net / maxDD / ret-DD | fold read |
|---|---|---|---|
| 2022-07 → 2023-07 | **+$503** / $514 / 0.98 | +$508 / $515 / 0.99 | ~tie (0.004 net +$5, maxDD −$1 worse) |
| 2023-07 → 2024-07 | **+$384** / **$471** / 0.82 | +$328 / $509 / 0.64 | **0.005 wins** (net +$56, maxDD $38 better) |
| 2024-07 → 2025-07 | −$223 / $606 / −0.37 | −$223 / $606 / −0.37 | identical (gate didn't differentiate) |
| 2025-07 → 2026-06 | **−$162** / **$455** / −0.36 | −$188 / $480 / −0.39 | **0.005 wins** (loses $26 less, maxDD $25 better) |
| **aggregate net** | **+$502** | **+$425** | **0.005 by +$77** |

**maxDD:** 0.004 is equal-or-worse in all four folds (worse by $1 / $38 / equal / $25).

## Interpretation

- **The pre-registered acceptance bar** — "0.004 wins ONLY if net PnL ≥ 0.005 AND
  maxDD not worse, out-of-sample per fold" — is **not met**. 0.004 beats 0.005 on
  net in only one fold (by $5, with marginally worse maxDD), ties one, and **loses**
  the other two on both net and maxDD.
- **Classifier lift ≠ money edge.** This is the crux MB-20260701-001 set out to
  test: a head that is ~37% better on f1_volatile produces a *worse* gated book. A
  denser volatile label makes the gate fire more often (labels more bars volatile →
  more OFF-cell gating), and in this roster/window that extra gating trimmed slightly
  more good trades than bad — net-negative. The prevalence-sensitive classifier
  metric was the wrong proxy for the routing decision, exactly as gate 2 cautioned.
- **The book is regime-dependent, not threshold-dependent.** Both arms are strongly
  positive in the 2022-2024 folds and negative in 2024-2026 (the same shape the
  live gate shows) — the threshold barely moves either; fold 3 is byte-identical.
  The vol-gate's value is the OFF-cell suppression it already does at 0.005, not a
  finer volatile label.

## Decision / status — MB-20260701-001 RESOLVED (no live change)

- **Keep the live vol-gate threshold at 0.005.** The shipped `btc-regime-15m-lgbm-v2`
  advisory head stays; no promotion, no threshold flip, no config touched.
- **The operator-approved "flip" is declined on this evidence.** The approval was
  explicitly conditional on the gate-4 money A/B; the A/B says the flip loses money.
  (Holding for this A/B instead of flipping on the gate-2 classifier lift was the
  right call.)
- **Gate chain complete:** gate 1 (mapping reconcile) ✅ · gate 2 (operating curve) ✅
  · gate 4 (money A/B) ✅ NEGATIVE. Gate 3 (RG4 live-regime discrimination) is moot
  for a threshold change that the money A/B already rejects — it was only ever a
  precondition *for* a flip, and there is no flip.
- **Pins demoted back to candidate** after the A/B (they are offline research
  artifacts; shadow was only needed transiently so the backtest could load them).
- **Broader lesson (logged):** for any ML-verdict routing gate, the classifier metric
  is a screen, not the decision — the money A/B is authoritative. `MB-20260717-VOLGATE-CLASSIFIER-VS-MONEY`.
