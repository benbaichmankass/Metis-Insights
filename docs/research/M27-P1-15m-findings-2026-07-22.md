# M27 P1 — 15m timeframe sweep findings (2026-07-22)

**Question:** does the proven ict_scalp pipeline — config-exact backtest,
gross→net fees (7.5 bps round-trip taker), decision-time regime stamps against
frozen vol edges, per-(trend,vol) OFF-cell gate, anchored 4-fold k-fold OOS —
hold at the **15m** timeframe for BTC + the P0 crypto passers, and does 15m add
anything over each symbol's existing 5m cell?

**Answer: yes for the alts, no for BTC.** XRP/ETH/SOL clear the P1 gate ("net
expectancy > 0 with ≥3/4 folds positive") in a *baseline-or-off-cells*
configuration; AVAX passes marginally; **BTC and ADA are rejected at 15m**.
XRP is the standout — the only symbol whose baseline AND off-cells are both
every-fold-positive at 15m.

## Rig (the run of record)

- Data: the on-disk Bybit linear **5m** CSVs (2023-01-01 → 2026-07-20,
  ~373.5k bars/symbol; BTC fetched fresh this run — it was excluded from the
  Batch-1 pull) resampled to **native 15m** (UTC-aligned `15min` buckets,
  OHLCV) trainer-side. Resample chosen over a native-15m pull for a
  zero-new-dependency run; the 2023-only frozen vol spec is identical either
  way.
- Pipeline: `scripts/research/m27/run_symbol_p0.py --timeframe 15m` (the
  `--timeframe` passthrough merged in #7293 for Batch-4) → derives the FROZEN
  tercile vol edges from the **2023 calendar year only** → `backtest_ict_scalp.py
  --stamp-regime --sim-breakeven` (live-exit-faithful) → `kfold_oos.py --folds 4
  --fee-bps-roundtrip 7.5`. Live YAML params, live break-even-trail exit.
- Raw artifacts: trainer `/home/ubuntu/m27_out_15m/<SYMBOL>/{volspec_5m,
  volspec_15m,backtest,emit,kfold}.json`; driver log
  `/home/ubuntu/m27_out_15m_driver.log`; fold-level JSONs mirrored in relay
  issues #7383 / #7384.
- **Methodology caveat (native-15m input):** on a 15m CSV the driver derives
  both `volspec_5m` and `volspec_15m` from the same (15m) series, so `vol5 ==
  vol15` and the two-axis OFF-cell rule degenerates to single-axis. The
  `off_cells_5m` / `calm_only_5m` / `calm_only_15m` arms therefore collapse to
  the same single-axis gate here — expected for a native-timeframe run and
  consistent with Batch-4 XAUUSD-15m. Read the gate as "single vol-axis
  filter," not the full BTC two-axis shape.

not).

## Per-symbol read

- **XRPUSDT — the standout.** 4/4 baseline AND 4/4 off-cells; the single most
  robust 15m cell found. Consistent with XRP's 5m result (gate load-bearing,
  off-cells 4/4). Strong P4 candidate.
- **ETHUSDT / SOLUSDT — clean passes.** Both 3/4 baseline with a positive
  aggregate; ETH's single-axis gate lifts it to 4/4 (best expectancy 0.142),
  SOL's baseline is the stronger config (highest TotR at 15m). Both already
  trade 5m `live` on `bybit_1` paper — a 15m leg is an *additional* cell, not a
  replacement.
- **AVAXUSDT — marginal.** Clears ≥3/4 on baseline/off-cells/calm but
  expectancy is ~0.025 net — real but thin; treat as a watch/soak candidate,
  not a confident promote.
- **BTCUSDT — reject at 15m.** Baseline 2/4 (+0.012 exp) and off-cells
  net-negative. The 15m TF does not add over BTC's already-live 5m gated leg;
  no 15m cell warranted.
- **ADAUSDT — reject at 15m.** 2/4 baseline + negative off-cells, the same
  weakness ADA showed at 5m (BTC-shape gate does not transfer). No leg.
