## Divergences found

### 1. `backtest_system.py` registry drift (harness ≠ live)
`scripts/backtest_system.py::_STRATS` hardcodes `trend_donchian` at **tf "2h"**, but the
live `trend_donchian` runs at **1h**. The registry tf is stale. (Several others in the
registry are pre-multi-symbol-roster and predate the per-symbol trend/pullback cells.)

### 2. This session's P1 book used non-live params (caveat on P1)
The P1 book (`docs/research/P1-vol-target-result-2026-06-26.md`, relays #4674/#4675)
was assembled from harness-default runs that diverge from live:

| P1 cell | P1 used | Live truth | Verdict |
|---|---|---|---|
| trend BTC/ETH 2h | **2h, no conf gate** | 1h, **min_conf 0.6** | wrong TF + missing gate |
| trend MES 2h | 2h | MES is `mes_trend_long_1d` (1d) | wrong instrument/TF |
| fade BTC 4h | included as live | `fade_breakout_4h` is **shadow** | non-live sleeve |
| squeeze BTC 4h | kc_mult default (1.5) | **kc_mult 1.0** | param mismatch |
| pullback BTC 2h | ran it | **no live BTC pullback 2h** | spurious cell |
| pullback MES 2h | ran it | **no live MES pullback 2h** | spurious cell |
| pullback ETH 2h | generic 2h | `eth_pullback_2h` 2h tp_r 50 trail 5 | ~matches |
| fvg_range BTC 15m | 15m default | 15m min_conf 0.0 | matches |

**Implication:** P1's *baseline book magnitudes* (Sharpe 0.27; the per-sleeve attribution
incl. `fvg_range_BTC` −41R) are **NOT live-representative** — the book is a proxy with a
mis-timeframed/un-gated trend sleeve, a shadow strategy included, and two cells that
don't exist live. **The P1 OVERLAY CONCLUSION still holds** (vol-targeting improving a
vol-clustered return stream is a general property, robust to book composition), but the
numbers are indicative, not a live-book measurement. Re-running P1 on the live param set
is the clean follow-up (`PB-20260626-005`).
