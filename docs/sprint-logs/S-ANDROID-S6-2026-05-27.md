## Work Completed
- **Android PR [`ict-trader-android#16`](https://github.com/benbaichmankass/ict-trader-android/pull/16)** — `feat(s6): trade-log widget — scrollable 7-day closed-trade history`. 4×5 default. `TradeLogRefreshWorker` pulls `/api/bot/trades/closed?limit=200`, filters client-side to the last 7 days, caches JSON in SharedPrefs. Glance `LazyColumn` renders one row per trade (close time / symbol / side pill / signed-USD P&L / close-reason badge). Also added the **refresh ↻ button** to both this widget AND the existing status widget — `ActionCallback` enqueues a `OneTimeWorkRequest` of the periodic worker so the operator gets immediate refresh without waiting for the next tick. Merged at `b0f9599`.
- **Android PR [`ict-trader-android#17`](https://github.com/benbaichmankass/ict-trader-android/pull/17)** — `feat(s6): candle-chart home-screen widget`. 5×3 default. `CandleChartRefreshWorker` pulls `/api/bot/candles?symbol=BTCUSDT&interval=15m&limit=60`, hands the candles to `CandleChartRenderer` (pure `android.graphics.Canvas` drawing — wick + body per candle, 4 horizontal grid lines with price labels, title overlay), PNG-compresses to `${cacheDir}/candle_widget.png`. The widget composable reads the PNG synchronously inside `provideGlance` and renders via Glance's `Image(ImageProvider(bitmap))`. Refresh ↻ included from day one. Merged at `86a193a`.
- **Android PR [`ict-trader-android#15`](https://github.com/benbaichmankass/ict-trader-android/pull/15)** — `fix: auto-register FCM token on rotation + cold-start sweep`. Operator reported (paraphrased): "I haven't gotten a push notification since v2 of the app." Root cause: the FAD-loop uninstall + reinstall gave the app a fresh FCM token, but `onNewToken` only logged to logcat — never POST'd to `/register`. The bot's `device_tokens` row kept its stale token, FCM responded `UNREGISTERED`/`INVALID_REGISTRATION`, `FcmNotifier._publish_one` swallowed the failure (intentional observer-only contract) → operator silently stopped getting pushes. **Fix:** new `FcmDeviceRegistration.attempt()` helper that fetches the current FCM token + POSTs to `/api/bot/devices/register`. Wired into both `IctMessagingService.onNewToken` (canonical FCM rotation callback) AND `MainActivity.onCreate` (defensive sweep for users who were on the pre-fix build when the rotation happened). Merged at `84077dd`.

## Risks and Follow-Ups
- Remaining technical risks:
  - **No on-device verification yet of the candle chart.** A render bug (e.g. coordinate math, missing label) would only surface when the operator places the widget. Mitigation: the renderer falls back to "no candles" / "flat range" / "Bot unreachable" text in degenerate cases, so the worst case is "ugly placeholder" rather than crash.
  - **Server-side stale-token scrub not yet shipped.** Even with the Android-side auto-register, a device that's been uninstalled (no future cold start) will linger as a dead row in `device_tokens` and waste `_publish_one` calls forever. The clean fix is a server-side `_publish_one` enhancement that DELETEs the row on FCM `UNREGISTERED` / `INVALID_REGISTRATION` responses. Logged as a follow-up PR.
