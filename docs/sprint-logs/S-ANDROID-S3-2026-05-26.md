## Contradictions or Drift Found
- **App version naming convention is unsettled.** S1 chose `versionName = "0.1.0-s1"`, S2 lifted it to `"0.1.0-s2"`. The FAD fix in S3 made versionName a runtime-suffixed string (`"0.1.0-s2.<run-number>"` in CI; `"0.1.0-s2"` locally). This means the on-device About card now shows two different versionNames depending on whether the operator installed a CI build (most common) or a manual `./gradlew assembleDebug`. Not a contradiction per se, but the next sprint should decide whether to advance to `0.1.0-s3` or keep stamping the channel as `0.1.0-s2.<N>` until M12 ships a first non-debug build.
- **Bottom-nav information density.** The S3 reshuffle put Health under Status (5 surfaces, all live data) and Debug under Settings (config + pairing tools). Logical, but it means a stale screenshot from S2 (with the old Debug tab) won't match the current shape — future sprint logs / S6 widget docs should not paste old screenshots without re-capturing.

## Risks and Follow-Ups
- Remaining technical risks:
  - **The operator's phone is still on the buggy versionCode=1 build until they uninstall.** Until then, FAD will keep prompting on every launch and the install will keep no-opping. Mitigated by the runbook step in PR #12's body; not a code risk.
  - **`HttpLoggingInterceptor` is at `Level.BASIC` for the OkHttp client.** That logs the URL + status of every request to logcat. On a phone shared with non-operator users (e.g. lost device), the URLs include the bot's bearer-protected paths, but the bearer itself is in the `Authorization` header which `Level.BASIC` does NOT log. Headers would only leak at `Level.HEADERS` / `Level.BODY` — and we're not at those. Calling this out so future tuning doesn't accidentally turn body-logging on.
- Remaining product decisions (Tier 3): none from S3. M12 stays out-of-tier for the trading pipeline.
- Blockers: none for S4 — the bot side already has the FCM notifier (M12 S1 wrap, PR #2055/#2075/#2082) and the Telegram→FCM mirror is alive. S4's job is the Android-side subscription UI + the bot-side per-event-kind filtering at notifier dispatch time.
