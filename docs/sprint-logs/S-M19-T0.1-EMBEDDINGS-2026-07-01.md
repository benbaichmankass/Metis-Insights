# Sprint Log: S-M19-T0.1-EMBEDDINGS-2026-07-01

## Date Range
2026-07-01 (single session, continued from the M19 roadmap design session).

## Objective
Ship **M19 T0.1** — the pretrained-TSFM embedding feature block: the cheapest,
first taste of the "reads-everything" representation idea. Turn a frozen
`amazon/chronos-bolt-tiny` (9M, CPU) into a feature extractor whose pooled
encoder embedding, projected to a fixed width, becomes optional `tsfm_emb_*`
columns on `market_features`, and stand up the offline A/B (embedding vs
no-embedding) on the BTC 15m regime head. Offline-only, no live-path change.

## Tier
**Tier 1** (trainer-side ML tooling: new dataset feature block + producer script
+ a `candidate`-stage manifest + tests + a trainer-side optional dep). No
`config/` (live), `src/` order-path, or unit-file change. The manifest stays at
`candidate` → refused by the live shadow factory → never order-influencing.

## Starting Context
M19 was proposed the same day (design doc + roadmap milestone, PR #5268). The
operator approved proceeding with T0.1 as the first sprint. Key established
facts: the binding constraint is labels, not compute; small TSFMs run CPU-only
and expose `embed()`; and the existing optional-feature-block pattern
(`cross_asset`/`macro`/`funding`/`orderflow`) is exactly what a new representation
block plugs into.

## Work Completed
- **`ml/datasets/embedding_features.py`** (new) — stdlib-only at import time
  (`ml.*` imports without torch/numpy). Defines `EMBEDDING_FEATURE_COLUMNS`
  (`tsfm_emb_0..31`), a **seeded, data-independent Gaussian random projection**
  (JL — deterministic, leakage-free reduction of the `d_model` pooled embedding
  to 32), a **lazily-imported, injectable** Chronos embedder (`chronos_embed_fn`
  + `embed_available`), and the pure producer `compute_embedding_feature_rows`
  (past-only trailing window per bar, `stride` + as-of carry-forward to bound
  cost, neutral `0.0` on short window / embedder failure — a missing
  representation never crashes a build).
- **`ml/datasets/families/market_features.py`** — wired the optional
  `embedding_path` side-stream exactly like `cross_asset` (import + schema +
  `iter_rows` kwarg + as-of load/align + row merge), bumped `builder_version`
  v9→v10, documented the kwarg. `0.0` on every row when omitted → every existing
  build byte-identical.
- **`scripts/ml/build_embeddings.py`** (new) — the producer (mirrors
  `build_cross_asset.py`): reads a `market_raw` dir, runs the frozen TSFM over
  each strided bar's trailing window, writes the `tsfm_emb_*` side-stream +
  metadata. Guards `--out-dim == EMBEDDING_DIM` and a clear "install
  requirements-backtest.txt" error when the deps are absent.
- **`ml/configs/btc-regime-15m-lgbm-emb-v1.yaml`** (new) — the A/B treatment leg:
  identical to `btc-regime-15m-lgbm-v2` except it adds the 32 `tsfm_emb_*`
  columns; `target_deployment_stage: candidate` (offline only); reads
  dataset `v003` (built with `embedding_path`). Baseline control = the v2 head on
  v002 (base columns are identical regardless of `embedding_path`).
- **`requirements-backtest.txt`** — added `chronos-forecasting` + `torch`
  (CPU/ARM), **trainer-side only**, with a comment that the live money-box never
  installs them (T0.1 stays candidate; the block lazy-imports).
- **`tests/ml/test_embedding_features.py`** (new, 16 tests) — column contract,
  projection determinism/shape/zero-input/data-independence, strided-index +
  min-context emission, fixed width, embedder-failure→neutral, **past-only
  window proof**, and a `market_features` integration test (with-vs-without
  `embedding_path`), all via an injected stub embedder (no torch needed).
- Updated `test_market_features.py::test_builder_version_is_v10` for the bump.

## Risks and Follow-Ups
- **The A/B itself is not yet run** — it needs a trainer-VM run (install the deps,
  build the `v003` embedding dataset, train both legs, `python -m ml compare`).
  That's the deliverable evidence; queued as the next step (trainer-vm-diag /
  model-training).
- **Chronos `embed()` API surface** is exercised only on the trainer (CI has no
  torch); the injected-stub tests cover all block logic, but the real embedder
  path (`chronos_embed_fn`) is validated when the trainer run happens — verify the
  pooled-embedding shape there and adjust `mean(dim=1)` if the API differs.
- **Per-bar Chronos cost** is bounded by `stride` + as-of fill; confirm the build
  wall-clock on the 1-OCPU trainer and tune `stride` if needed.
- **Random projection is a v0 reduction** — PCA/learned reduction (fold-aware, no
  leakage) is a documented follow-up if T0.1 shows lift.

- T0.2 (unsupervised regime discovery), T0.3 (conviction stacker + embeddings),
