## Work Completed
- **feat(overview)** (`2874725`): `OverviewScreen` mirroring Streamlit's
  Overview — status pill, live chart (symbol/interval chips, `/api/bot/candles`
  via the shared `CandleChartRenderer` bitmap path with position/signal/
  closed-trade overlays + live PnL), KPI row, 24h scorecard + VM health,
  open positions, per-strategy 24h. Default home destination.
- **fix(orderpackages)** (`158657a`): root cause — `ClaudeScore.{entryQuality,
  exitQuality,riskManagement}` typed `Double` but the bot serves categorical
  strings (`score_order_packages.py`); any scored package broke the whole
  `/api/bot/order-packages` parse. Verified: `db_info` showed **1421**
  order_packages rows present (#2254). Modelled as `String?`, rendered as labels.
- **fix(accounts)** (`c808f7a`): added `getConfig()` and drove the account
  list from `/api/bot/config.accounts[]` (matching Streamlit) instead of the
  balance/activity union — so IBKR `ib_paper` (no snapshot when the Gateway
  is logged out) + the dry prop account always show. LIVE/DRY pill, venue,
  strategies, halt banner; falls back to the old enumeration on config outage.
- **fix(widget,performance)** (`4e47a51`): W-1 — removed the outer `Column`
  `clickable(actionStartActivity)` that shadowed the chip/↻ `ActionCallback`s
  in Glance; open-app target moved to non-interactive sections. Performance —
  separate manual-refresh flag so pull-to-refresh shows a spinner.
- **fix(network)** (`009ea76`): audit-driven (subagent cross-checked all 21
  `BotApi` data classes vs routers). `ClosedTrade.closedAt` → `String?`
  (trades_closed.py serves null when a closed trade has no linked
  order_package + no `notes.closed_at`); `RegisterDeviceResponse.subscriptions`
  → `JsonElement?` (the bot can echo a dict shape).
- **fix(mobile-push)** (`ebee84b`): `notifier._build_message` now sets
  `android.priority=HIGH` so FCM delivers data messages immediately instead
  of batching them out of Doze.
- **fix(liveness-watchdog)** (`66dad08`): new `--boot-grace-seconds` (env
  `LIVENESS_BOOT_GRACE_SECONDS`, default 0 = off). Within N s of host boot
  (`/proc/uptime`, fail-open to `+inf`) a stale/missing heartbeat is
  suppressed along with autoheal, and `last_status` is not marked stale → no
  "recovered" ping when the trader returns; a still-stale heartbeat past the
  window alerts as a genuine failure-to-recover. Unit enables it at 600 s.
- **docs(roadmap)** (`e284045`): corrected the M12 Overview overclaim + added
  the S9 entry.
- PRs opened (draft): `ict-trader-android#38`, `ict-trading-bot#2258`.
