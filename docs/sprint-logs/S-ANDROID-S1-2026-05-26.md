# Sprint Log: S-ANDROID-S1

## Date Range
- Start: 2026-05-26
- End: 2026-05-26 (split across two sessions — the second session is wrapping up)

## Objective
- Primary goal: Prove the bot→FCM→Android push-notification pipe end-to-end. The S1 deliverable is a single-screen debug app on the operator's phone that receives a notification within seconds of a paper-trade close on `bybit_1`.
- Secondary goals:
  - Lay the bot-side foundation (notifier module, `device_tokens` table, registration endpoint) so S2–S7 can build on it without touching live-path code again.
  - Open the `benbaichmankass/ict-trader-android` repo and ship the project scaffold so any subsequent contributor (or session) can open it in Android Studio and build.
  - Wire `FCM_SERVICE_ACCOUNT_JSON` rotation onto the `system-actions` ladder so the operator never has to SSH-edit the trader's `.env`.

## Tier
- Tier 2 (dominant) — the trade-closed observer hook in `Database.update_trade`, plus `set-mobile-push-secrets` / `enable-mobile-push` / `disable-mobile-push` actions, all touch the live VM's `.env` or the close write-path.
- Tier 1 supporting work — the notifier module, device-registration router, sprint plan, runbook, sprint log, ROADMAP update.
- Out-of-tier — everything in `benbaichmankass/ict-trader-android` (separate codebase; no money-at-risk surface).
- Justification: Anything that wraps the close-write path was wrapped in `try/except`, feature-flagged on `MOBILE_PUSH_ENABLED` (default off), and verified to never propagate into the trader. The notifier holds no decision authority.
