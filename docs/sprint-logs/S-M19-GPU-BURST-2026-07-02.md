## Work Completed
Landed as 6 small PRs (all squash-merged to `main`), each fixing the next
real-world blocker uncovered by a cheap live run:

1. **PR #5448** — dropped a `%` from the pod SSH bootstrap: RunPod interpolates
   `docker_args` raw into its GraphQL mutation, and `printf "%s\n"` raised
   `Syntax Error: Unexpected character: "%"` before any pod launched.
2. **PR #5450** — dropped the custom `docker_args` entirely: the next char (`$` in
   `$PUBLIC_KEY`) broke GraphQL too. Root fix — the official `runpod/pytorch:*`
   image already injects `PUBLIC_KEY` into `authorized_keys` and runs `sshd` on 22,
   so we pass only `env={"PUBLIC_KEY": pub}` + `support_public_ip` + `ports="22/tcp"`
   and let the image do it. Zero operator config, no shell-in-GraphQL fragility.
3. **PR #5452** — **the on-pod train driver**: replaced the rc5 "not yet wired"
   stub with the real run, reconciled against the ACTUAL `ml` CLI (verified vs
   `run_training_cycle.sh` + `build_trainer_datasets.sh`). `build-dataset` IS real
   (a passthrough to `ml.datasets`); `train --export-onnx --parity-gate` is
   FICTIONAL (no ONNX/parity anywhere in the codebase). The driver reads the
   manifest's `dataset:` block (single source of truth), **hard-refuses any
   non-`market_features` crypto manifest before a byte leaves the VM** (data
   contract), pins the SHA, runs `build-dataset market_raw`→`market_features`→
   `ml train` over SSH, and returns a `{model_state, metrics, manifest}` JSON
   bundle (the LightGBM booster, not ONNX). `--manifest`/`--max-minutes` plumbed
   through `run_burst.sh` + the workflow (new `manifest:` issue field); the bundle
   uploads as a workflow artifact (no multi-MB git blob).
4. **PR #5454** — pod `pip install` aborted on `Cannot uninstall blinker 1.4`
   (distutils package pip can't uninstall). First (over-broad) fix: global
   `--ignore-installed`.
5. **PR #5456** — SSH keepalive (`ServerAliveInterval=30`/`ServerAliveCountMax=20`/
   `TCPKeepAlive=yes`) so a long, quiet training run isn't idle-dropped
   (`client_loop: send disconnect: Broken pipe`, rc=255).
6. **PR #5458** — two of my own regressions: (a) the global `--ignore-installed`
   reinstalled all 25 deps (~10-15 min) and ate the `max_minutes` budget →
   surgical `--ignore-installed blinker` then a plain `-r requirements.txt` (fast,
   reuses the image's wheels); (b) `_ssh_capture`'s `TimeoutExpired` handler did
   `(e.stderr or "") + "\n(timeout)"`, but `TimeoutExpired` carries bytes even
   under `text=True` → `TypeError` that crashed the driver and hid the partial
   train log → added `_as_text()`.
