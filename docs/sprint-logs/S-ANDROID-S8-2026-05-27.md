### Widget polish (operator-driven follow-ups)
- **PR #28** Dashboard widget 5×7 + edge-to-edge chart + chips below — outer column padding removed for the chart section, per-section `SECTION_PAD = 8.dp` on status + trade log, `ChartImageBox` 120dp → 240dp. Operator preference for maximum chart real estate on the home screen. Merged.
- **PR #29** Smart y-range extension so overlay layers actually appear — `CandleChartRenderer` now extends lo/hi to include overlay prices (positions' entry/SL/TP, latest signal FVG/sweep) within ±150% of the candle span, falling back to the silent-drop behaviour when further. Guarantees candles always occupy ≥40% of the chart height (`1 / (1 + 1.5)`). Plus chart bars 15 → 25. Merged.
- **PR #30** Chip toggle now repaints DashboardWidget + 35 bars — every refresh worker (`CandleChartRefreshWorker`, `StatusRefreshWorker`, `TradeLogRefreshWorker`) now calls `DashboardWidget().updateAll(ctx)` in addition to its own standalone widget. Bars 25 → 35. Merged. **Operator-reported residual:** chips + ↻ still don't fire on the dashboard widget — see W-1 below; the fix was structurally necessary but not sufficient.

## Risks and Follow-Ups
- Remaining technical risks:
  - **W-1 widget chip/refresh regression.** The chip taps on the unified dashboard widget appear to fire MainActivity (the outer clickable) instead of the per-chip ActionCallback. Easy fix (remove outer clickable, attach per-section clickables on the non-interactive areas) but not yet shipped. Currently the operator can still use the chips on the standalone candle-chart widget; only the unified dashboard widget is affected.
  - **Data Explorer wide-table rendering.** The vertical key=value approach degrades when a table has many columns + many rows — every cell becomes a separate `Text`. Acceptable for 20 rows × ~10-30 columns but won't scale to large pages. Out of scope for v1.
- Remaining product decisions (Tier 3): none. M12 stays out-of-tier for the trading pipeline throughout.
