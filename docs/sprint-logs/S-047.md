## Work Completed

### T1 — accounts.yaml routing (D2)
- `bybit_2` declared as `market_type: spot-margin` in `config/accounts.yaml`.
- Risk-manager spot-margin parameters (`max_borrow_btc`, `borrow_fee_apr_pct`, `liquidation_buffer_pct`) live in the risk-rule configuration surface with sensible hardcoded defaults.

### T2 — RiskManager spot-margin sizing (D3)
- `RiskManager.position_size` computes spot-margin sizing using USDT collateral, liquidation-buffer, and borrow-fee budget.

- Daily-loss-budget rule still wins on conflict.
- Unit tests cover all four refusal arms.

### T3 — execute.py + coordinator wiring (D4 + D5)

- `coordinator.py` direction-aware balance update follows.
- Bybit testnet smoke (`scripts/sprint047/spot_margin_smoke.py`) opens a sell with `isLeverage=1`, verifies the borrow line appears in `get_wallet_balance`, closes it, verifies the borrow line clears.
