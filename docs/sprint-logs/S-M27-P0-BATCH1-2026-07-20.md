### New infra finding — trainer-vm-diag drops the SSH channel on long-silent commands

Three consecutive dispatches (issues #7297/#7300/#7301) hit an SSH
`Broken pipe` at 2-8 minutes into a silent (no-stdout) foreground command —
independent of backgrounding style (plain foreground, `nohup &` + poll,
`timeout` wrapper). Only a **fully detached** launch
(`setsid nohup ... > log 2>&1 < /dev/null & disown -a`, returning from the
SSH command almost immediately) reliably avoided it (#7302, confirmed — the
178k-bar backtest that appeared to hang for 8+ min actually completed in
under 3 minutes once run detached). Also confirmed the free-text prose
*after* an issue's `cmd:` block is fed to bash as part of the same script
(currently harmless — it reliably fails as shell syntax — but latent).
Logged as `BL-20260721-TRAINERDIAG-SILENT-SSH-DROP` (health-review backlog),
not fixed this session (touches a shared workflow). Composes with the
existing `BL-20260720-TRAINERDIAG-FENCE`.
