# S-015 T3 — Harness validation on repo fixtures

_Generated by `scripts/sprint015/analyze_fixtures.py`. **This is harness validation, not strategy quality.** The fixtures committed to `data/` cover only a few days each — orders of magnitude too small for real backtest claims. The 5-year analysis lands in Session B once the data fetcher has outbound network access._


## Goal

Demonstrate the T1 harness running end-to-end against existing repo OHLCV fixtures. Confirm:

1. The data pipeline (CSV → DataFrame → resample → strategy) is wired and produces sensible-shape output.
2. The slippage sensitivity dial works in both directions.
3. The signal density responds monotonically to the threshold parameter (sanity check on the strategy adapter).
4. Hour-of-day attribution buckets are produced for the eventual killzone overlay analysis.
